## Новый функционал Python-скрипта (test_bd.py)

### Терминальный интерфейс (CLI)
Меню из 7 пунктов без графических элементов:

| Пункт | Функция |
|-------|---------|
| 1 | Просмотр всех запчастей |
| 2 | Добавление запчасти |
| 3 | Поиск (по OEM, марке/модели, году) |
| 4 | Сортировка (Mergesort) |
| 5 | Поиск через дерево (ОПД А2) |
| 6 | Удаление запчасти |
| 7 | Редактирование запчасти |

### Сортировка Mergesort
Реализована классическая сортировка слиянием. Доступные поля для сортировки:
- **Цена** — числовая сортировка
- **OEM-номер** — строковая сортировка
- **Год выпуска** — числовая сортировка
- **Состояние** — строковая сортировка (new/used)

**Опциональность:** можно выбрать одно поле или несколько (сортировка применяется последовательно).

### Дерево оптимального поиска (ОПД А2)
Построение сбалансированного бинарного дерева поиска по **OEM-номерам**.

**Алгоритм:**
1. Выгрузка всех записей из БД
2. Сортировка по OEM-номеру
3. Построение сбалансированного BST (медиана в корне)
4. Рекурсивное построение левого и правого поддеревьев

**Сложность поиска:** O(log n) вместо O(n) при линейном переборе.

**Использование:**
```
Пункт 5 → введите OEM-номер → поиск в дереве
```

# Основные SQL-запросы

## Получение всех запчастей:
```sql
SELECT * FROM parts_inventory ORDER BY id;
```

## Поиск по OEM-номеру:
```sql
SELECT * FROM parts_inventory 
WHERE oem_number ILIKE '%51220-S04-003%';
```

## Поиск по авто (марка + модель + год):
```sql
SELECT * FROM parts_inventory 
WHERE brand ILIKE '%Honda%' 
  AND model ILIKE '%CR-V%'
  AND year_start <= 2000 
  AND year_end >= 2000;
```

## Фильтрация по состоянию и наличию:
```sql
SELECT * FROM parts_inventory 
WHERE condition = 'used' AND quantity > 0;
```

# Логика работы алгоритмов

## Mergesort (сортировка слиянием)
```
1. Разделение списка пополам до единичных элементов
2. Слияние с сортировкой по заданному ключу
3. Поддержка сортировки по нескольким полям последовательно
```

## ОПД А2 (оптимальное бинарное дерево поиска)
```
1. Сортировка записей по OEM-номеру
2. Выбор медианы в качестве корня
3. Рекурсивное построение левого поддерева (элементы < медианы)
4. Рекурсивное построение правого поддерева (элементы > медианы)
5. Поиск: сравнение с узлом → переход влево/вправо → повтор
```
